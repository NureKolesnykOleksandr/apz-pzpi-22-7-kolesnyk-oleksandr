Business Requirements (Бізнес рішення)
1.1.	Background(Передумови)
	У сучасному світі зростаюча кількість носимих пристроїв створює нові можливості для моніторингу здоров'я в режимі реального часу. Швидкий розвиток технологій IoT (Internet of Things) та носимих сенсорів відкриває перспективи для створення персоналізованих систем охорони здоров'я, які можуть значно покращити якість життя людей та допомогти в запобіганні серйозним захворюванням.
	Сучасне суспільство стикається з численними викликами у сфері охорони здоров'я: зростання кількості хронічних захворювань, старіння населення, перевантаження медичних закладів та потреба в своєчасному виявленні критичних станів здоров'я. Особливо актуальною є проблема відсутності постійного моніторингу життєво важливих показників для людей похилого віку та осіб з хронічними захворюваннями.
	Програмна система моніторингу здоров'я "MedMon" спрямована на вирішення цих проблем, надаючи користувачам можливість безперервно відслідковувати свій стан здоров'я через інтеграцію з носимими пристроями. Система забезпечує збір та аналіз даних з різних сенсорів, включаючи пульс, рівень кисню в крові, температуру тіла, активність та фази сну, що дозволяє виявляти аномалії на ранніх стадіях та своєчасно реагувати на критичні зміни.
	Головна ідея розробки — створити централізовану автоматизовану систему, яка дозволить користувачам отримувати актуальні дані про свій стан здоров'я, аналізувати тенденції та отримувати персоналізовані рекомендації для покращення якості життя. Метою розробки є забезпечення зручного та ефективного інструменту для профілактичного моніторингу здоров'я, який допоможе запобігти серйозним захворюванням та сприятиме формуванню здорового способу життя.
	Такий продукт буде корисним для широкого кола користувачів: від звичайних громадян, які піклуються про своє здоров'я, до людей з хронічними захворюваннями, медичних працівників та родичів, які хочуть відслідковувати стан здоров'я своїх близьких. Система дозволить не лише відстежувати поточні показники, але й впроваджувати превентивні заходи для підтримання здоров'я, що є важливим аспектом у розбудові здорового суспільства.
1.2.	Business Opportunity (Бізнес можливості)
	Ринок носимих пристроїв і цифрових рішень для охорони здоров'я демонструє стабільне зростання, створюючи унікальні можливості для розробки інноваційних систем моніторингу здоров'я. За останні роки попит на персоналізовані рішення для контролю здоров'я значно зріс, особливо серед людей похилого віку та осіб з хронічними захворюваннями.
Основними викликами в цій галузі є недостатня доступність комплексних рішень для тривалого контролю стану здоров'я, своєчасного виявлення потенційних загроз і оперативного реагування на критичні ситуації. Існуючі на ринку продукти часто мають обмежену функціональність або високу вартість, що робить їх недоступними для широкого кола користувачів.
Аналіз існуючих рішень показав наступні недоліки:
	Wearable Fitness Trackers забезпечують базове відстеження активності та параметрів здоров'я, включаючи кроки та пульс, але не підтримують функції глибокого аналізу даних або персоналізованих рекомендацій.
	Remote Patient Monitoring Systems орієнтовані на медичні установи, але їх використання у повсякденному житті складне через високу вартість і необхідність професійного обслуговування.
	Mobile Health Applications пропонують обмежений функціонал для користувачів, але не інтегруються з носимими пристроями чи сенсорами, що зменшує їхню ефективність у моніторингу в реальному часі.
	Emergency Notification Systems фокусуються на оповіщеннях про надзвичайні ситуації, але не мають аналітичного модуля для виявлення потенційних ризиків.
Переваги програмного продукту "MedMon":
	Комплексний підхід об'єднує сучасні сенсорні технології, інтелектуальний аналіз даних та інтуїтивно зрозумілий веб-інтерфейс для забезпечення повноцінного моніторингу здоров'я.
	Доступність продукту орієнтована на широкий ринок, включаючи приватні домогосподарства, людей з хронічними захворюваннями та осіб похилого віку, на відміну від багатьох комерційних систем.
	Персоналізація системи генерує індивідуальні рекомендації на основі аналізу особистих даних користувача, враховуючи його вік, медичні показники та цілі.
	Превентивний підхід фокусується на раннє виявлення аномалій та запобігання критичним станам, а не лише реагування на вже наявні проблеми.
	Безпека даних гарантує захист персональних медичних даних користувачів відповідно до сучасних стандартів конфіденційності.
Цільова аудиторія включає наступні сегменти користувачів:
	Люди з хронічними захворюваннями, зокрема діабет, гіпертонія та серцево-судинні захворювання, які потребують постійного моніторингу критичних показників.
	Особи похилого віку та їхні родичі, які хочуть забезпечити безпеку та контроль стану здоров'я старших членів сім'ї.
	Активні користувачі, які піклуються про своє здоров'я та прагнуть оптимізувати свій спосіб життя на основі об'єктивних даних.
	Медичні працівники, які можуть використовувати систему для моніторингу пацієнтів між візитами до клініки.
Модель монетизації передбачає:
	Freemium модель надає базову версію продукту безкоштовно з обмеженим функціоналом, що дозволяє користувачам ознайомитися з можливостями системи.
	Преміум підписка включає розширену аналітику, персоналізовані рекомендації, інтеграцію з додатковими сенсорами та автоматизовані звіти.
	Корпоративні рішення представляють спеціальні пакети для медичних установ та страхових компаній з додатковими функціями управління та звітності.
Ринок цифрової охорони здоров'я активно розвивається, оскільки суспільство стає більш свідомим щодо профілактики захворювань та потребує ефективних рішень для підтримання здоров'я. Зважаючи на зростаючі вимоги до якості життя та демографічні зміни, цей продукт матиме стабільний попит як у приватних користувачів, так і в корпоративному та медичному секторах.
2.	Vision of the Solution (Концепція та рішення)
2.1.	Vision Statement (Окреслення концепції)
	Програмна система моніторингу здоров'я "MedMon" призначена для забезпечення ефективного та безперервного контролю стану здоров'я користувачів, сприяння формуванню здорового способу життя та запобіганню серйозним захворюванням. Цей продукт створюється для задоволення потреб широкого кола користувачів — від звичайних громадян до людей з хронічними захворюваннями — у простому та доступному рішенні для моніторингу життєво важливих показників через інтеграцію з носимими пристроями та IoT-технологіями.
	Система надаватиме комплексний аналіз даних з різних сенсорів (пульс, рівень кисню в крові, температура тіла, активність, фази сну), генеруватиме персоналізовані рекомендації, забезпечуватиме своєчасні попереджувальні сповіщення та можливості налаштування індивідуальних параметрів моніторингу. Завдяки інтеграції з сучасними носимими пристроями, система буде автоматично збирати та обробляти дані про стан здоров'я користувача в режимі реального часу.
		Бачення продукту полягає у створенні екосистеми, де моніторинг здоров'я стане доступним, ефективним та зрозумілим для всіх. Програмна система "MedMon" змінить підхід до профілактичної медицини, дозволяючи кожному користувачеві — від активної молодої людини до пацієнта з хронічними захворюваннями — мати точну і постійну інформацію про свій стан здоров'я. У світі, де цей продукт стане повсюдно доступним, люди зможуть приймати обґрунтовані рішення щодо свого здоров'я, своєчасно виявляти потенційні проблеми та адаптувати свій спосіб життя для досягнення оптимального самопочуття.
	Продукт орієнтований на створення персоналізованого досвіду для кожного користувача. Для звичайних громадян система дозволить легко контролювати основні показники здоров'я та отримувати рекомендації для покращення якості життя. Для людей з хронічними захворюваннями продукт запропонує поглиблений моніторинг критичних параметрів та автоматичне оповіщення лікарів або близьких у разі виявлення загрозливих змін. Для медичних працівників система стане інструментом для віддаленого моніторингу пацієнтів та отримання об'єктивних даних про їхній стан між візитами.
	Концепція побудована на принципах:
	Превентивність — фокус на попередженні захворювань, а не лише їх лікуванні
	Персоналізація — індивідуальний підхід до кожного користувача
	Доступність — простота використання для людей різного віку та рівня технічної грамотності
	Безпека — надійний захист персональних медичних даних
	Інтеграція — сумісність з різними типами носимих пристроїв та медичного обладнання
	Зважаючи на очікуване зростання попиту на цифрові рішення в охороні здоров'я та технології превентивної медицини, цей продукт стане важливою складовою сучасної екосистеми здоров'я, сприяючи як покращенню індивідуального благополуччя користувачів, так і зменшенню навантаження на систему охорони здоров'я в цілому.
2.2.	Major Features
	MF-1: Збір даних з сенсорів Система забезпечує автоматичний збір даних з носимих пристроїв, що відслідковують ключові показники здоров'я: пульс, рівень кисню в крові (SpO2), температуру тіла, рівень фізичної активності та фази сну. Інтеграція з різними типами сенсорів дозволяє створити комплексну картину стану здоров'я користувача в режимі реального часу.
	MF-2: Аналіз отриманих даних та виявлення аномалій Інтегрована аналітична система обробляє зібрані дані, використовуючи алгоритми машинного навчання для виявлення відхилень від нормальних показників користувача.
	MF-3: Генерація персоналізованих рекомендацій На основі проаналізованих даних система створює індивідуальні рекомендації для покращення здоров'я та самопочуття. Рекомендації можуть включати поради щодо фізичної активності, сну, харчування та інших аспектів здорового способу життя, адаптовані під конкретного користувача.
	MF-4: Веб-інтерфейс для доступу до історичних даних Користувачі мають доступ до зручного веб-інтерфейсу, через який можна переглядати поточні показники, історію змін та рекомендації.
	MF-5: Налаштування персональних сповіщень Система дозволяє користувачам налаштовувати індивідуальні параметри для отримання сповіщень про аномальні показники, досягнення цілей або потенційні ризики. Користувачі можуть встановлювати порогові значення для різних параметрів та обирати способи отримання повідомлень.
	MF-6: Оповіщення про критичні стани У разі виявлення критичних відхилень у показниках здоров'я система автоматично надсилає екстрені повідомлення попередньо визначеним контактам (близьким, лікарям, службам екстреної допомоги). Це забезпечує швидке реагування на потенційно небезпечні ситуації.
	MF-7: Реєстрація та управління обліковими записами Система підтримує функції реєстрації нових користувачів, авторизації та управління особистими профілями. Користувачі можуть налаштовувати свої медичні профіли, включаючи інформацію про хронічні захворювання, приймані ліки та особливості здоров'я.
	MF-8: Інтеграція з носимими пристроями Система забезпечує сумісність з широким спектром носимих пристроїв різних виробників через стандартизовані API та протоколи передачі даних. Це дозволяє користувачам обирати пристрої відповідно до своїх потреб та бюджету.
Ці функції роблять продукт унікальним і привабливим для користувачів, які прагнуть не лише відстежувати свої показники здоров'я, але й активно впроваджувати заходи для покращення якості життя та запобігання захворюванням. Комплексний підхід до моніторингу здоров'я відрізняє "MedMon" від існуючих фрагментарних рішень на ринку.
3.	Scope and Limitations (Рамки та обмеження)
3.1.	Scope of Initial Release (Рамки первинного випуску)
У початковому релізі система "MedMon" надаватиме базовий набір функцій для ефективного моніторингу стану здоров'я. Основна мета першого випуску — забезпечити стабільну роботу ключових можливостей, необхідних для запуску продукту на ринку та задоволення первинних потреб користувачів.
	Основні функції першого випуску:
	Автоматичний збір даних із сенсорів: пульс, рівень кисню в крові (SpO2), температура тіла, рівень фізичної активності (кроки, калорії), основні фази сну
	Збереження та структурування отриманих даних в базі даних
	Базова валідація та фільтрація даних для забезпечення їх достовірності
	Простий алгоритм виявлення аномалій на основі встановлених медичних норм
	Порівняння поточних показників з індивідуальними базовими значеннями користувача
	Виявлення критичних відхилень, що потребують негайного втручання
	Зручна панель управління для перегляду поточних показників здоров'я
	Простий перегляд історичних даних у вигляді базових графіків та таблиць
	Можливість налаштування порогових значень для персональних сповіщень
	Базове управління обліковим записом та налаштуваннями профілю
	Автоматичні сповіщення про критичні стани здоров'я через електронну пошту
	Попереджувальні повідомлення при перевищенні встановлених порогових значень
	Можливість налаштування списку екстрених контактів для оповіщення
	Система реєстрації нових користувачів з валідацією базових даних
	Безпечна авторизація з використанням стандартних методів аутентифікації
	Управління особистими профілями та зміна паролів
	Шифрування конфіденційної інформації при зберіганні та передачі
	Реалізація базових стандартів безпеки для захисту персональних медичних даних
	Контроль доступу на основі ролей (користувач/адміністратор)
	Технічна архітектура початкового релізу:
	.NET для обробки бізнес-логіки та API серверної частини
	SQLite для зберігання користувацьких даних та показників здоров'я
	Blazor з CSS для створення веб-інтерфейсу клієнтської частини
	Базові API для підключення популярних носимих пристроїв 
	Мобільний додаток для перегляду історичних даних.
	Обмеження початкового релізу:
	Система працює виключно в онлайн-режимі
	Підтримка обмеженої кількості типів носимих пристроїв
	Базовий аналіз даних без використання штучного інтелекту
	Обмежені можливості візуалізації та аналітики
Початковий випуск зосереджений на критичних функціях, необхідних для демонстрації цінності продукту та отримання зворотного зв'язку від користувачів, що дозволить оптимізувати розвиток системи в наступних версіях.
3.2.	Scope of Subsequent Releases (Рамки наступних випусків)
У наступних версіях системи MedMon планується розширити функціонал за рахунок інтеграції штучного інтелекту (ШІ) для аналізу даних і покращення якості моніторингу здоров’я. Основним фокусом буде прогнозування потенційних ризиків на основі показників сенсорів (пульс, рівень кисню, активність, сон) та надання персоналізованих рекомендацій через чат-бота. Для роботи ШІ система використовуватиме дані з носимих пристроїв користувачів, а також їхні налаштування (вік, медичні показники, цілі). Спочатку передбачається використання готових API-рішень для базового аналізу, а з часом – навчання власної моделі на зібраних даних. Також у майбутніх оновленнях буде додано розширену підтримку сенсорів, включаючи пристрої для вимірювання тиску та ЕКГ, а також інтеграцію з медичними сервісами для обміну даними з лікарями.

4.3. Operating Environment (Робоче середовище)
Система моніторингу здоров'я "MedMon" розробляється для роботи в сучасному технологічному середовищі з використанням передових веб-технологій та хмарних рішень. Архітектура системи орієнтована на забезпечення високої доступності, масштабованості та безпеки для користувачів з різних географічних регіонів.
Технологічний стек:
Серверна частина:
	Мова програмування: C# з використанням .NET 8.0+ для забезпечення кросплатформенності та високої продуктивності;
	Фреймворк: ASP.NET Core для створення RESTful API та обробки HTTP-запитів;
	ORM: Entity Framework Core для взаємодії з базою даних та управління даними;
IoT частина: 
	буде працівати з сенсорами температури, рівнем кисню у кров, активністю та фазами сну. Дані з цих сенсорів будуть оброблюватися та передаватися на сервер програмою, написаною мовою програмування C#;
База даних:
	Основна БД: SQLite для локального розгортання та тестування;
Клієнтська частина:
	Веб-інтерфейс: Планується створити з використанням Blazor (Blazor WebAssembly) з CSS для стилізації інтерфейсу;
	Мобільний застосунок: Передбачена розробка для платформи Android з використанням Kotlin..
